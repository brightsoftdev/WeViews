WeViews

Tutorial

Install

  • Download the distribution from here (zip, tarball).
  • Add only the "WeViews" subdirectory to your project.
    Do not add the "WeViewsDemo" subdirectory to your project.

Core Concepts

  • All UIViews have a "natural" size. This is the size returned by [UIView sizeThatFits:].
    For an UIImageView, its the size of its image. For a (non-wrapping) UILabel, its the size of its text in its font.
  • FPanels re-layout their contents whenever a) their size is changed with [setFrame:] or [setBounds:] or b) [FPanel layoutContents] is invoked.

Demo App

  • The Demo app allows you to build and tinker with user interfaces interactively. Playing with it is a good way to get a feel for how the library works.
  • To run the demo, open the WeViewsDemo Xcode project.
  • The Demo requires iOS 3.2+ and is best run in the iPad Simulator.
  • Add views to the canvas using the "Add Views" sidebar. New views are added to the currently selected view or layer.
  • Select views by clicking on them in the canvas or by selecting them in the "View Hierarchy" sidebar.
  • You can inspect and modify view & layer properties using the "Selection" sidebar.
  • One useful feature of the Demo app is that it can generate example code for any UI you have built. Select any view and use the "Generate Code" link in the "Selection" sidebar.
